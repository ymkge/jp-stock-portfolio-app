# 市場指標サマリー表示機能 基本設計書 (最新版)

## 1. 機能概要
ポートフォリオのメインページおよび分析ページの上部に、市場全体の動きを把握するためのサマリーカードを表示する。日経平均、TOPIX、および日経平均先物の現在値、前日比に加え、過去の履歴データに基づいた前週比（WoW）、前月比（MoM）を可視化する。

## 2. データソースと監視対象
Yahoo!ファイナンスの以下のエンドポイントからデータを取得する。

| 指標名 | コード | 用途 |
| :--- | :--- | :--- |
| **日経平均株価** | `998407.O` | 国内市場の代表的な現物指数 |
| **TOPIX** | `998405.T` | 市場全体の広範な動きを示す現物指数 |
| **日経平均先物** | `5040469.O` | 大阪取引所の中心限月（1限月）。先行指標 |

## 3. バックエンド設計

### 3.1 スクレイピングロジック (`scraper.py`)
- **新設クラス**: `IndexScraper` ( `BaseScraper` を継承)
- **解析パス**: 指標ページ特有の構造に対応するため、以下の優先順位で探索を行う。
    1.  `data["mainDomesticIndexPriceBoard"]["indexPrices"]` (現物指数・先物の主要パス)
    2.  `data["mainIndicesPriceBoard"]["indices"]` (複数指標リスト内の合致検索)
- **URL生成**: 個別銘柄用の `.T` 付与は行わず、コードをそのまま使用。
- **資産タイプ**: `market_index` として定義し、個別銘柄用の分析ロジックから分離。

### 3.2 履歴・比較ロジック (`history_manager.py` / `app.py`)
- **過去データの特定**: `get_historical_data_before(code, date_str)` を実装。
    - SQL: `SELECT ... WHERE code = ? AND date <= ? ORDER BY date DESC LIMIT 1`
    - 効果: 指定日が休業日（土日祝）であっても、その直近の営業日データを自動的に取得。
- **比較指標の計算**:
    - **DoD (前日比)**: Yahoo!ファイナンスの取得値をそのまま表示。
    - **WoW (前週比)**: 現在値と「7日前以前の最新値」を比較。
    - **MoM (前月比)**: 現在値と「30日前以前の最新値」を比較。

## 4. フロントエンド設計

### 4.1 UIコンポーネント
- **配置**: `templates/index.html` および `analysis.html` のメインコンテンツ最上部に `#market-summary-container` を新設。
- **カードデザイン**: 横並びレイアウト。各カードは指標名、現在値、前日比、前週比、前月比を表示。
- **視覚効果**:
    - 上昇: `price-up` クラス（赤色）
    - 下落: `price-down` クラス（緑色）
- **リンク**: 各カードをクリックすると Yahoo!ファイナンスの詳細ページを別タブで開く。

## 5. 設定管理 (`highlight_rules.json`)
- 新設キー `market_indices` にて、監視対象のコードと名称を管理。
- 将来的な指標の追加を、コード修正なしで可能にしている。

## 6. デグレ防止と安全性
- **完全分離**: 既存の `JPStockScraper` には一切触れず、専用の `IndexScraper` を追加。
- **障害隔離**: 指標取得に失敗しても API 全体はエラーにせず、該当指標のみ `--` 表示でフォールバック。
- **負荷軽減**: 指標取得も既存の並列処理制限（セマフォ）および市場時刻キャッシュロジックに統合。
